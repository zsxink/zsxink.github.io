# 第一个章节跳转被遮挡问题 - 解决方案

## 问题描述

在文章页面 `http://localhost:4322/posts/helloworld` 中，点击第一个章节链接时，章节标题会被导航栏遮挡，而点击第二个章节链接则正常。

## 根本原因分析

### 1. 第一个章节的特殊性

- 第一个章节标题（第一个h2元素）是页面中首个主要标题
- 由于CSS `mt-12`（上边距12个单位）的影响，`offsetTop`计算可能不准确
- 当页面滚动位置接近顶部时，激活项计算逻辑与滚动位置不同步

### 2. 技术冲突

- **JavaScript滚动脚本**：使用 `target.offsetTop - 96px` 计算滚动位置
- **CSS滚动边距**：`scroll-margin-top: 96px` 作为回退
- **激活项计算**：`useActiveItem` 使用 `getBoundingClientRect().top <= 96` 判断
- 三者之间的计算基准不同，导致第一个章节位置计算不准确

### 3. 具体问题

1. 对于靠近页面顶部的第一个章节，`offsetTop - 96px` 可能得到接近0的值
2. `getBoundingClientRect()` 相对于视口，而 `offsetTop` 相对于文档
3. 当滚动位置在页面顶部时，激活项计算可能跳过第一个章节

## 解决方案

### 1. 改进JavaScript滚动计算

```javascript
// 使用更精确的getBoundingClientRect计算
const rect = target.getBoundingClientRect()
const currentScroll = window.scrollY || window.pageYOffset
const targetAbsoluteTop = currentScroll + rect.top
const targetPosition = Math.max(0, targetAbsoluteTop - headerHeight - extraPadding)
```

### 2. 增强CSS保障

```css
.markdown h1,
.markdown h2,
.markdown h3,
.markdown h4,
.markdown h5,
.markdown h6 {
  scroll-margin-top: 96px !important;
  scroll-padding-top: 96px !important; /* 添加scroll-padding作为额外保障 */
}

/* 确保第一个h2标题有足够的空间 */
.markdown h2:first-of-type {
  padding-top: 4px;
}
```

### 3. 修复激活项计算逻辑

```javascript
// 特殊情况：如果滚动位置在页面顶部，激活第一个标题
if (scrollY < 100) {
  const firstHeading = $headings[0]
  if (firstHeading && firstHeading.id) {
    setActiveItem(firstHeading.id)
    return
  }
}

// 使用更灵活的阈值
if (itemTop <= 120 && nextItemTop > 72) {
  setActiveItem(item.id)
}
```

### 4. 添加调试和边界检查

```javascript
// 添加详细的调试信息
console.log('滚动到章节:', targetId, {
  offsetTop: target.offsetTop,
  getBoundingClientRectTop: rect.top,
  currentScroll: currentScroll,
  targetAbsoluteTop: targetAbsoluteTop,
  targetPosition: targetPosition,
})

// 滚动完成后检查位置
setTimeout(() => {
  const finalRect = target.getBoundingClientRect()
  console.log('最终位置检查:', targetId, {
    topRelativeToViewport: finalRect.top,
    isVisible: finalRect.top >= headerHeight && finalRect.top < window.innerHeight,
  })
}, 100)
```

## 技术要点

1. **计算基准统一**：使用 `getBoundingClientRect()` + `window.scrollY` 而不是 `offsetTop`
2. **边界保护**：使用 `Math.max(0, ...)` 防止负值滚动位置
3. **多重保障**：JavaScript滚动 + CSS滚动边距 + 激活项计算修复
4. **调试友好**：详细的控制台输出便于问题排查
